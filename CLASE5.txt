Para integrar **Interaxia** con la API de Google Gemini utilizando Node.js y un frontend web, seguiremos una arquitectura cliente-servidor estándar.

### 1. Procedimiento para crear la aplicación

Para que tu frontend (HTML) se comunique con el backend (Node.js), necesitamos exponer un endpoint en el servidor que reciba las consultas del usuario y las envíe a la API de Gemini.

* **Entorno:** Usaremos `Express` para crear un servidor web sencillo.
* **Comunicación:** El frontend enviará una petición `POST` (usando `fetch`) al backend.
* **Seguridad:** La API Key de Google **nunca** debe estar en el HTML, siempre se gestionará en el servidor (backend) usando variables de entorno.

---

### 2. Pasos para crear el proyecto Node.js

1. **Crea una carpeta para el proyecto** y entra en ella:
```bash
mkdir interaxia-app && cd interaxia-app

```


2. **Inicializa el proyecto:**
```bash
npm init -y

```


3. **Instala las dependencias necesarias:**
```bash
npm install express @google/generative-ai dotenv cors

```


*(Nota: Usaremos la librería oficial actual `@google/generative-ai` para mayor compatibilidad).*
4. **Crea un archivo llamado `.env**` para tu API Key:
```env
API_KEY=tu_api_key_aqui

```



---

### 3. Procedimiento de integración

* **Backend:** Configura `cors` para permitir que tu `index.html` (que corre en el navegador) pueda hablar con tu servidor local (puerto 3000).
* **Frontend:** Utiliza la API `fetch` de JavaScript para enviar el mensaje del usuario al servidor y esperar la respuesta de la IA.

---

### 4. Código fuente de los componentes

#### A. Backend (`server.js`)

```javascript
require('dotenv').config();
const express = require('express');
const cors = require('cors');
const { GoogleGenerativeAI } = require("@google/generative-ai");

const app = express();
app.use(express.json());
app.use(cors());

const genAI = new GoogleGenerativeAI(process.env.API_KEY);

app.post('/api/chat', async (req, res) => {
    try {
        const { message } = req.body;
        const model = genAI.getGenerativeModel({ model: "gemini-1.5-flash" });
        const result = await model.generateContent(message);
        res.json({ response: result.response.text() });
    } catch (error) {
        res.status(500).json({ error: error.message });
    }
});

app.listen(3000, () => console.log('Servidor Interaxia corriendo en http://localhost:3000'));

```

#### B. Frontend (`index.html`)

```html
<!DOCTYPE html>
<html>
<body>
    <h2>Interaxia Chat</h2>
    <input type="text" id="userInput" placeholder="Escribe algo...">
    <button onclick="sendMessage()">Enviar</button>
    <div id="response"></div>

    <script>
        async function sendMessage() {
            const message = document.getElementById('userInput').value;
            const res = await fetch('http://localhost:3000/api/chat', {
                method: 'POST',
                headers: { 'Content-Type': 'application/json' },
                body: JSON.stringify({ message })
            });
            const data = await res.json();
            document.getElementById('response').innerText = data.response;
        }
    </script>
</body>
</html>

```

### Ejecución

1. En tu terminal, ejecuta: `node server.js`
2. Abre el archivo `index.html` en tu navegador.

El error que estás experimentando se debe a dos causas técnicas claras:

1. **Error 500 (Internal Server Error):** Esto significa que tu backend en Node.js está fallando al procesar la petición. Probablemente se deba a que la variable de entorno `API_KEY` no se está leyendo correctamente o falta una configuración en el servidor.
2. **CORS/Security Origin:** Al abrir tu archivo HTML directamente haciendo doble clic (o con F5 en VS Code), el navegador lo interpreta como `file:///...`. Los navegadores bloquean las peticiones AJAX desde archivos locales por seguridad.

### Solución paso a paso

#### 1. Corregir el error 500 (Backend)

Es posible que `dotenv` no esté cargando el archivo en la carpeta correcta. Asegúrate de que `server.js` y el archivo `.env` estén en el mismo directorio. Modifica temporalmente tu `server.js` para ver qué está pasando exactamente:

```javascript
// En tu server.js, dentro del bloque catch:
app.post('/api/pregunta', async (req, res) => {
    try {
        const { pregunta } = req.body;
        if (!process.env.API_KEY) throw new Error("API Key no configurada");
        const model = genAI.getGenerativeModel({ model: "gemini-1.5-flash" });
        const result = await model.generateContent(pregunta);
        res.json({ respuesta: result.response.text() });
    } catch (error) {
        console.error("Detalle del error:", error); // Esto aparecerá en tu terminal
        res.status(500).json({ error: error.message });
    }
});

```

*Mira la terminal después de intentar enviar de nuevo; ahí aparecerá el error real.*

#### 2. Corregir el error de origen (Frontend)

No abras `index.html` directamente. Debes servirlo a través de un servidor local.

* **Si usas Visual Studio Code:** Instala la extensión **"Live Server"**.
* En lugar de presionar F5, haz clic derecho sobre `index.html` y selecciona **"Open with Live Server"**.
* Esto abrirá tu web en `http://127.0.0.1:5500` (o similar). Al estar en un entorno `http`, el navegador permitirá que el `fetch` se comunique con tu servidor en `http://localhost:3000`.

### Resumen de validación

* **Servidor:** Asegúrate de que el archivo `.env` tenga el formato exacto: `API_KEY=AIzaSy...` (sin comillas).
* **Frontend:** Asegúrate de que tu script esté llamando exactamente a `http://localhost:3000/api/pregunta`.
* **Terminal:** Siempre revisa los logs de tu terminal (donde corre `node server.js`) cada vez que intentes una consulta; ahí verás el "Internal Server Error" detallado.

Si el error persiste, por favor comparte el mensaje exacto que aparece en la terminal de Node.js después de realizar la consulta desde el navegador.
